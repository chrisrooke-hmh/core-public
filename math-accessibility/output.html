Math Equations

Below is a collection of different type of math equations, the equations are in KaTeX, mathML and MathJax, there is minimal styling here the purpose of this page is to identify different accessibility needs across the different math equation tools.

Two different sources have been used to genererate mathML equations.

  • Temml A TeX-to-MathML conversion library which converts latex to MathML
  • MathJax which converts tex to MathML.

Findings from Dev experience

The mathJax versions of mathMl come with more semantic information than the MathML generated by Temml. This is because mathJax supports adding TeX hints to mathML and adding the original TeX source as an annotation, (these must be enabled however and are not turned on by default)

The Temml comes with some base styling, similiar to katex, however the mathJax does not. For the purpose of this POC, the temml styling is applied to the mathJax equations to ensure visual consistency around typography.

While styling is applied through css, some limitations still exist, for example all the borders must be added by the author, or a suite of css classes must be created and the author would still need to apply these classes, alternatively, there may be a formattor or equation generator that will do this. Temml generated the inline styles for the mathML, while mathJax does not, For the purpose of this poc, the mising styles have been left out for now, these missing styles are present in the mathJax version of the mathML which is below the Temml version.

Per MDN "this text can become very verbose for complex content and so requires proper authoring tools such as converters from a lightweight markup language or WYSIWYG equation editors". Some of these tools can be found here or through the use of editors like TinyMCE

No math is natively supported in selects, there is an example with the math visually rendering correctly however without the use of javascript or some plugin, it is not possible to match the visual dispaly with the actual read out versions

Anyhwere where there is a MathML equation there are two versions, the temml version and the mathjax version with the temml version coming first.

Pros vs Cons

KaTeX

Pros
  • Compatibility: Works well with modern web technologies and integrates easily with JavaScript frameworks.
  • Ease of Use: Simple to use with KaTeX syntax, which is familiar to many users.
  • Styling: Provides good control over the styling of mathematical expressions.
Cons
  • Accessibility: Limited accessibility features compared to MathML. Screen readers may not interpret KaTeX-rendered content as effectively.

MathMl

Pros
  • Accessibility: Excellent accessibility support. Screen readers and other assistive technologies can interpret MathML content effectively.
  • Standardization: MathML is a W3C standard, ensuring broad compatibility and future-proofing.
  • Integration: Integrates well with other XML-based technologies and can be embedded directly in HTML.
Pros
  • Browser Support: Legacy Browsers may be an issue
  • Styling: Limited control over styling compared to KaTeX and MathJax.

MathJax

Pros
  • Feature-Rich: Supports a wide range of KaTeX features and extensions, making it suitable for complex mathematical content.
  • Accessibility: Strong accessibility support, with features designed to work well with screen readers.
  • Customization: Highly customizable with extensive configuration options.
Cons
  • Size: Larger footprint, which can impact load times and performance.
